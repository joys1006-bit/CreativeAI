# 🖥️ 프론트엔드 개발 워크플로우

## 담당 에이전트
- **실행**: 프론트엔드 개발자(L4), FE UI 개발자(L3)
- **설계**: 프론트엔드 아키텍트(L6)
- **디자인 검수**: UI 디자이너(L3), CDO(L7)
- **최종 승인**: CPO(L7)

## Step 1: 요구사항 분석
// turbo
1. CPO 관점에서 사용자 가치 확인
2. CDO 관점에서 디자인 시스템 정합성 확인
3. 영향 범위 파악 (어떤 컴포넌트/CSS에 영향?)

## Step 2: 설계
1. 프론트엔드 아키텍트 관점에서 컴포넌트 구조 결정
2. CSS 클래스명 규칙 확인:
   - 컴포넌트 JSX에서 사용하는 클래스 ↔ CSS 파일 정의 **1:1 매칭 필수**
   - 새 클래스 추가 시 기존 클래스와 충돌 여부 검증
3. 상태 관리 방식 결정

## Step 3: 구현
// turbo
1. code-convention 스킬의 프론트엔드 컨벤션 준수
2. Import 순서: 외부 → 전역 → 공통 → 페이지 → CSS
3. 한글 주석 필수
4. 컴포넌트명 = 파일명 (PascalCase)

## Step 4: CSS 검증 (⚠️ 필수)
// turbo
1. JSX에서 사용하는 **모든 className**이 CSS에 정의되어 있는지 확인
2. 미디어쿼리에만 정의되고 기본 스타일이 없는 클래스 확인
3. CSS 변수(var(--xxx)) 정의 여부 확인
4. 8px 그리드 준수 여부 확인

## Step 5: 브라우저 검증
1. 개발 서버 실행 (`npm run dev` 또는 `npx vite`)
2. 화면 스크린샷 또는 브라우저 확인
3. 반응형 레이아웃 검증 (모바일/태블릿/데스크톱)

## Step 6: 검수 및 배포
1. quality-gate 체크리스트 통과
2. `/deploy` 워크플로우 실행
