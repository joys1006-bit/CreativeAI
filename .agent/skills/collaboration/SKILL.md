# 🤝 에이전트 협업 및 의사결정 스킬 (Collaboration Skill)

## 🎯 개요
이 스킬은 서로 다른 전문성을 가진 에이전트들이 충돌 없이 협업하고, 프로젝트의 전역적인 일관성을 유지하기 위한 상호 자문 및 합의 규칙을 정의합니다.

## 📏 협업 핵심 룰 (Core Rules)

### 1. 전역 변경 시 상호 자문 필수
- **API 명세 변경**: 백엔드 아키텍트가 API를 변경하기 전, 프론트엔드 아키텍트의 컨슈머 사이드 검토를 반드시 거쳐야 합니다.
- **데이터 스키마 변경**: 백엔드 로직 개발자가 DB 필드를 수정할 때, DBA의 성능/정합성 심사를 받아야 합니다.
- **UI 컴포넌트 라이브러리 추가**: 프론트엔드 개발자가 서드파디 라이브러리를 도입할 때, 디자인 팀의 디자인 시스템 정합성 확인을 받아야 합니다.

### 2. 의사결정 투명성
- 모든 에이전트 간의 합의 사항은 `master_handbook.md`의 **PART 4**에 기록하여 모든 에이전트가 즉시 학습하도록 합니다.

### 3. 직무 범위를 넘어서는 제안
- 에이전트는 자신의 주 직무가 아니더라도 타 분야 에이전트에게 '전문가적 관점의 개선 제안'을 할 수 있으나, 최종 결정은 해당 분야 에이전트의 권한을 존중합니다.

## 🛠️ 협업 프로세스 (Skill Logic)
1. **제안**: 변경이 필요한 에이전트가 다른 에이전트에게 검토 요청.
2. **논의**: 연관된 전문가들이 각자의 페르소나 관점에서 피드백 제공.
3. **합의**: 최적의 대안을 선택하고 기록.
4. **실행**: 합의된 내용을 코드로 구현.
