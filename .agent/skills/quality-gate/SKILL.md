# ✅ 품질 게이트 스킬 (Quality Gate Skill)

## 🎯 개요
모든 작업 완료 시 통과해야 하는 품질 검증 체크리스트입니다. QA 엔지니어가 관장하며, 모든 에이전트가 자체 점검 기준으로도 활용합니다.

## 🚦 게이트 판정 기준
- **🟢 PASS**: 모든 필수 항목 통과 → 배포 가능
- **🟡 CONDITIONAL**: 필수 통과, 권장 일부 미충족 → 리더 판단 후 배포
- **🔴 FAIL**: 필수 항목 1개 이상 미통과 → 배포 불가, 수정 후 재검수

---

## 📋 공통 체크리스트

### [필수] 코드 품질
- [ ] code-convention 스킬의 명명 규칙 준수
- [ ] 사용하지 않는 코드(Dead Code) 제거
- [ ] 한글 주석 포함 (핵심 비즈니스 로직)
- [ ] 에러 핸들링 누락 없음

### [필수] 보안
- [ ] security-protocol 스킬의 커밋 전 체크리스트 통과
- [ ] 하드코딩된 시크릿 없음
- [ ] 인증/인가 검사 적용

### [필수] 정합성
- [ ] 변경된 파일 간의 참조 관계 정상 (import, className ↔ CSS 등)
- [ ] 빌드 에러 없음
- [ ] 기존 기능 회귀(regression) 없음

### [권장] 프론트엔드 추가
- [ ] JSX className ↔ CSS 정의 1:1 매칭 확인
- [ ] 반응형 레이아웃 검증 (데스크톱/모바일)
- [ ] 접근성(a11y) 기본 요소 확인

### [권장] 백엔드 추가
- [ ] 단위 테스트 작성/통과
- [ ] API 응답 형식 일관성
- [ ] 로깅 적절성 (DEBUG/ERROR 레벨)

---

## 🛠️ 실행 가이드
1. 작업 완료 시 위 체크리스트를 자체 점검
2. [필수] 항목 전부 통과 확인
3. 미통과 항목이 있으면 수정 후 재점검
4. 결과를 보고서에 포함하여 리더에게 보고
