# 🔍 코드 및 상호 검수 스킬 (Code Review Skill)

## 🎯 개요
고품질의 코드와 디자인 결과물을 유지하기 위한 상호 검수 기준과 절차를 정의합니다.

## 📏 검수 기준 (Review Standards)

### 1. 개발자 페르소나 검수
- **Clean Code**: 이름의 명확성, 함수의 단일 책임 원칙 준수 여부.
- **Robustness**: 예외 처리 누락 여부, 리액티브 스트림의 에러 전파 확인.
- **TDD Compliance**: 기능 구현 전/후로 유닛 테스트가 동반되었는지 확인.

### 2. 디자이너 페르소나 검수 (UI Linting)
- **Pixel Perfection**: 8px 그리드 준수 여부, 컬러 토큰 일치 여부.
- **Motion Integrity**: 프레임워크 애니메이션의 일관성 및 자연스러움 점검.

### 3. 보안/QA 페르소나 검수
- **Endpoint Security**: 권한 검증(Auth) 누락 여부.
- **Corner Cases**: 사용자 입력값의 엣지 케이스 처리 여부.
- **Security Protocol**: security-protocol 스킬 체크리스트 통과 여부.

## 🏅 점수 판정 기준

| 등급 | 점수 | 의미 |
|------|------|------|
| **S (Excellent)** | 90-100 | 모범 코드, 즉시 배포 가능 |
| **A (Good)** | 75-89 | 소수 개선사항, 배포 승인 가능 |
| **B (Acceptable)** | 60-74 | 수정 필요, 조건부 승인 |
| **C (Needs Work)** | 40-59 | 상당한 수정 필요, 배포 불가 |
| **F (Rejected)** | 0-39 | 반려, 전면 재작업 |

### 채점 항목 (각 20점)
1. **기능 정확성** — 요구사항 충족, 버그 없음
2. **코드 품질** — 컨벤션 준수, 가독성, 유지보수성
3. **보안** — 시크릿 노출, 인증 검사, 입력 검증
4. **정합성** — 타 모듈과의 호환, CSS-JSX 매칭
5. **문서화** — 한글 주석, 변경 이력, 보고 품질

## 🛠️ 실행 가이드

### 자체 검수 프로세스
1. 작업 완료 후 quality-gate 스킬 체크리스트 자체 점검
2. 위 5개 채점 항목으로 자체 점수 산출
3. B등급(60점) 이상이면 리더에게 검수 요청
4. C등급 이하면 자체 수정 후 재검수

### 교차 검수 프로세스
- 에이전트는 작업을 마친 후, 연관된 타 에이전트에게 결과물을 공유하고 '전문가 판정(Approved/Requested Changes)'을 받도록 유도합니다.
- **FE 작업** → UI 디자이너 + 프론트엔드 아키텍트 검수
- **BE 작업** → 백엔드 아키텍트 + CISO 보안 검수
- **보안 관련** → CISO 필수 검수
